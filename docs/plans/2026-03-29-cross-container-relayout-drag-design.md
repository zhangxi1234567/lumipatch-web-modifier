# Cross-Container Relayout Drag Design

**Date:** 2026-03-29

## Goal

Replace the current mixed `fixed` / same-container drag behavior with a true page relayout mode that lets users grab a visible element, move it across containers, and drop it into a new structural position without triggering the element's click behavior.

## User Requirements

- Any visible page element can be dragged as if it were furniture in a room.
- Dragging must make surrounding content reflow instead of stacking elements on top of each other.
- Dropping must work across containers, not only within one list or flex parent.
- A drag gesture must not also activate the dragged element's native click behavior.
- After moving, the original element should keep its semantics and actions; if it is a home link before dragging, it is still the same home link after dragging.
- Saved moves should reapply automatically on future visits.

## Recommended Approach

Use a real DOM reparenting drag model.

During drag, the original node becomes a floating preview while a visible placeholder is inserted into the live document flow. As the pointer moves, the placeholder is moved between candidate containers and insertion points so the page continuously reflows around the intended drop location. On drop, the original node is inserted where the placeholder sits and the placeholder is removed.

Persistence should no longer store `position: fixed` coordinates. Instead, it should store a structural move rule describing which node moved, which parent now owns it, and which sibling it should appear before. The content script should replay those move rules before other DOM mutations.

## Why This Approach

- It matches the requested “redecorate the room” mental model.
- It preserves the dragged node and its event handlers instead of cloning content.
- It supports cross-container reordering while keeping page semantics intact.
- It avoids the current overlay-style stacking problem.

## Rejected Approaches

### CSS-only fixed positioning

Rejected because it cannot make surrounding content yield space and collapses on responsive pages.

### Same-parent reorder only

Rejected because the user explicitly wants cross-container movement.

## Architecture Changes

### Content Script

- Replace free-position persistence with placeholder-driven DOM relocation.
- Add drop-container and insertion-point heuristics based on `elementsFromPoint()` and ancestor scanning.
- Add click suppression for completed drag gestures.
- Save structural move rules instead of CSS position rules.

### Rule Engine

- Add a new `moveNode` rule type.
- Normalize, merge, and describe `moveNode` rules alongside the existing rule types.

### Background

- Update `rules:save-drag` so it can save structural drag rules.
- Preserve backward compatibility with old style-value drag payloads if present.

## Drag Model

### Drag Start

- Resolve a stable drag target from the pointer-down element.
- Insert a visible placeholder into the current DOM position.
- Convert the real node into a floating drag preview.

### Drag Move

- Determine the best drop container from the pointer stack.
- Compute the insertion point within that container.
- Move the placeholder to that exact structural position.

### Drop

- Insert the real node where the placeholder sits.
- Remove temporary drag styles.
- Save a `moveNode` rule with the node selector, target parent selector, and optional `beforeSelector`.
- Suppress the click generated by the drag release.

## Selector Strategy

Use the existing selector builder as the base identifier for dragged nodes and containers. This remains heuristic and may be imperfect on highly dynamic apps, but it is consistent with the rest of the extension. `beforeSelector` should be omitted when the node is appended to the end of the target container.

## Risks

### Dynamic frontend re-renders

Frameworks may recreate containers and invalidate selectors. Mitigation: keep selectors as stable as possible and reapply on mutations through the existing observer.

### Illegal nesting

Some containers should not accept arbitrary children. Mitigation: use conservative drop-container heuristics and reject self/descendant insertion.

### Click leakage

Browsers can emit `click` after drag release. Mitigation: add a capture-phase click suppressor for the drag gesture completion.
